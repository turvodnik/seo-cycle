# Загрузка ключей в Keys.so clustering через браузер (runbook)

Keys.so clustering-инструмент НЕ доступен через API (только UI). Этот runbook —
полуавтоматическая загрузка через браузер (Claude in Chrome MCP / Codex browser skill).

> ⚠ **Дорого по токенам** (каждый шаг = снимок страницы в контекст) и хрупко (UI меняется).
> Применять для **больших ядер** (тысячи ключей), где SERP-кластеризация Keys.so окупает
> расход. Для малых/средних ядер — наша кластеризация (`03-clusters.md`, делает LLM) дешевле
> и достаточна. Требует залогиненную сессию Keys.so в браузере.

## Предусловие
1. Подготовь файл ключей (дёшево, без браузера):
   ```bash
   python3 ~/.claude/skills/seo-cycle/scripts/keyso-clustering-export.py \
     --from-keyso-cache <домен> --out <cycle>/keys-for-clustering.txt
   # или из нескольких источников: --from-md 02-keywords.md --from-csv ядро.csv keyword
   ```
2. Убедись, что в браузере открыта залогиненная сессия keys.so.

## Шаги (Chrome MCP / Codex browser)
1. `list_connected_browsers` / `select_browser` — выбрать браузер с сессией Keys.so.
2. `navigate` → раздел кластеризации в кабинете keys.so (Инструменты → Кластеризация).
3. Создать новый проект кластеризации: задать имя (напр. «emwoody <тема> <дата>»), базу (msk = Яндекс Москва), порог группировки (стандартный hard/soft по задаче).
4. `file_upload` → загрузить `keys-for-clustering.txt` (по ключу на строку).
5. Запустить кластеризацию, дождаться завершения (может занять минуты — проверять статус, не висеть в цикле; вернуться позже).
6. Экспортировать результат (CSV/Excel) или снять `get_page_text` с таблицы кластеров.
7. Сохранить результат в цикл: `<cycle>/03-clusters-keyso.md` (или .csv). Сверить/дополнить нашу `03-clusters.md`.

## Когда НЕ нужно
- Ядро < ~1000 ключей → наша кластеризация (LLM по SERP-overlap + интентам) дешевле и быстрее.
- Нужна только группировка для контент-плана → уже есть в `03-clusters.md`.

## Когда оправдано
- Большое ядро (тысячи ключей), где ручная/LLM-группировка непрактична.
- Нужна именно SERP-based кластеризация Keys.so (их алгоритм по реальной выдаче Яндекса) как второе мнение к нашей.

## Результат
`<cycle>/03-clusters-keyso.md` — кластеры от Keys.so. Объединить с нашей кластеризацией;
расхождения — повод перепроверить группировку приоритетных кластеров.
