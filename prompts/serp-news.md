# Универсальный промпт: что нового в SERP по теме

Phase 9 (monitoring) и Phase 10 (iteration) — отслеживание трендов, новых статей конкурентов, изменения SERP за период.

## Через что запускать

- **Perplexity Pro** (фильтр Time → Last 14 days)
- **Codex CLI** с web search (`codex exec`)

## Шаблон

```
Я отслеживаю SEO-нишу: {{niche}}, рынок {{region}}.
Покажи изменения по теме {{topic}} за последние {{N_days}} дней.

Конкретно:

1. **Новые материалы в топ-10 поисковиков** ({{engines}}) с {{дата - N дней}}
   по сегодня по запросам:
   - "{{главный ключ}}"
   - "{{второй ключ}}"
   - "{{третий ключ}}"
   Дай 5-8 URL с краткой аннотацией: чем сильна статья, какие H2-блоки, формат
   (статья / категория / товар), бренд автор.

2. **Новые PAA / Related Questions** — вопросы, которых не было N дней назад
   по нашим ключам. Минимум 10.

3. **Технологические / нормативные обновления** — новые {{normative_docs}}, изменения
   требований. Свежие новости от производителей / поставщиков ({{key_brands}}) —
   новые линейки, изменения составов / условий.

4. **Запросы с растущим объёмом** — что становится популярнее по Trends за период.

5. **Сезонные акценты** для текущего месяца (что в пике, входит, выходит).

Источники — обязательно ссылки. Без общих фраз — только конкретика.
```

## Плейсхолдеры

| Плейсхолдер | Из конфига |
|---|---|
| `{{niche}}` | `industry.name` |
| `{{region}}` | `locale.region` |
| `{{engines}}` | `engines[].name` через запятую |
| `{{N_days}}` | задаётся при запуске (обычно 14) |
| `{{normative_docs}}` | по нише |
| `{{key_brands}}` | из `industry.primary_categories` + реестр сущностей |

## Темы для регулярного мониторинга (cadence)

- Топ-категории / topic clusters сайта — раз в 2 недели
- Pillar-статьи блога — раз в месяц
- Сезонные темы — еженедельно в пиковом окне

## Что делать с результатами

- **Новые материалы конкурентов** → анализ что есть/нет у нас → в `10-iterations.md`
- **Новые PAA** → расширение FAQ через повторный publish
- **Нормативные обновления** → fact-check база, обновление `fact_check_log`
- **Растущие запросы** → новые spoke-статьи в content plan
- **Сезонные акценты** → корректировка cadence публикаций
