# Универсальный промпт: извлечение сущностей и интентов (Шестаков)

Используется в Phase 2 (keyword research) и Phase 4 (Entity Map) — наполнение реестра сущностей и карты интентов под конкретную тему.

## Через что запускать

- **Perplexity Pro** (браузер) — лучший: с источниками
- **Antigravity + Codex параллельно** через `llm-cli-collect.sh` — fallback без браузера
- **NeuronWriter** даёт SERP-based entities, но без интентов — комбинируй

## Шаблон

```
Я строю Entity Map по методике Олега Шестакова (Rush Agency) для SEO-оптимизации
страницы {{domain}}. Тема: {{topic}}.
Рынок: {{region}}, сегменты: {{target_audiences}}.
Язык контента: {{language}}.

Помоги собрать:

1. **Центральная сущность** — определение из 2-3 предложений (короткое, фактическое,
   подходит для AEO-цитаты в выдаче).

2. **Атрибуты центральной сущности** (10-15 пунктов): измеримые параметры с
   диапазонами значений (например: для физического товара — плотность, размеры,
   класс; для услуги — время, цена, охват; для софта — фичи, ограничения).

3. **Связанные сущности** — 15-20 ближних соседей по графу:
   - Контексты использования
   - Сопутствующие сущности (что покупают/применяют вместе)
   - Конкурирующие сущности (альтернативы)
   - Бренды / производители / поставщики
   - Региональные модификаторы (нормативы, локальные особенности)

4. **Тройки отношений** (subject-predicate-object) — минимум 12 явных формулировок
   для попадания в Knowledge Graph поисковика.

5. **Карта интентов** — отдельно:
   - **Явные** (что человек прямо ищет): commercial / informational / navigational / transactional
   - **Скрытые** (страхи, сомнения, «что если…»): минимум 7 пунктов
   - Под каждый интент — пример поискового запроса

6. **PAA / People Also Ask** — 15 реальных вопросов из выдачи по теме с короткими
   фактическими ответами (2-3 предложения каждый).

7. **Источники** — 3-5 авторитетных ссылок ({{normative_docs}}, научные публикации,
   гайды производителей) для fact-check.

Запрет: общие/оценочные слова («качественный», «надёжный», «эффективный»).
Только факты с цифрами и нормативами.
```

## Плейсхолдеры

| Плейсхолдер | Из конфига |
|---|---|
| `{{domain}}` | `project.domain` |
| `{{topic}}` | задаётся при запуске |
| `{{region}}` | `locale.region` |
| `{{target_audiences}}` | `target_audiences` |
| `{{language}}` | `locale.language` |
| `{{normative_docs}}` | по нише (см. fact-check.md) |

## Куда применять результаты

- **Сущности и связи** → реестр в `<entities_root>/entities.yaml`
- **PAA** → FAQ-репитеры страниц
- **Скрытые интенты** → spoke-статьи в content plan
- **Тройки отношений** → раздел 5 Entity Map шаблона
- **Источники** → внутренняя fact-check база (`fact_check_log`)
