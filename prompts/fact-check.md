# Универсальный промпт: fact-check

Используется в Phase 6 (writing) и Phase 10 (iteration) — проверка достоверности технических утверждений в контенте.

## Через что запускать

- **Perplexity Pro** (браузер) — лучший вариант, даёт URL источников
- **Codex CLI** (`codex exec`) — с live web search, fallback если нет Perplexity
- **Antigravity** (`agy`) — быстрый но без URL, использовать только для нетехнических утверждений

## Шаблон (заполняй `{{плейсхолдеры}}` под свой проект)

```
Я редактор технических статей про {{niche}} (рынок {{region}}, язык {{language}}).
Проверь, насколько достоверно следующее утверждение, и приведи 3 авторитетных
источника ({{normative_docs}}, ТУ производителя, профильные технические гайды,
научные публикации):

<УТВЕРЖДЕНИЕ>
"{{утверждение из черновика}}"
</УТВЕРЖДЕНИЕ>

В ответе:
1. **Оценка**: достоверно / частично достоверно / недостоверно / спорно.
2. **Точная формулировка** — как утверждение должно звучать корректно.
3. **3 источника** с прямыми URL и краткими цитатами.
4. **Контекст** — есть ли нюансы для {{region}} / целевой аудитории.
5. **Рекомендация для текста**: оставить, переформулировать (как именно), удалить.

Не используй общих слов («надёжный», «эффективный») — только конкретные цифры и нормативы.
```

## Плейсхолдеры (заполняй из `seo-cycle.yaml`)

| Плейсхолдер | Источник в конфиге | Пример |
|---|---|---|
| `{{niche}}` | `industry.name` | «стройматериалы» |
| `{{region}}` | `locale.region` | «Москва+МО» |
| `{{language}}` | `locale.language` | «русский» |
| `{{normative_docs}}` | по нише, см. ниже | «ГОСТ, СП, СНиП, СанПиН» |

### Нормативные документы по нишам (помогает Perplexity искать точнее)

- **Стройматериалы (РФ):** ГОСТ, СП, СНиП, СанПиН, ТУ производителей
- **Медицина / БАДы:** МКБ-10, Pharmaceutical Pharmacopoeia, ВОЗ guidelines, FDA monographs
- **Финансы:** ЦБ РФ, Налоговый кодекс, Минфин, ФНС
- **Юридические темы:** УК РФ, ГК РФ, КоАП, Постановления Пленума ВС
- **Автомобили:** Правила ЕЭК ООН, ТР ТС, инструкции производителя
- **Электроника:** IEEE standards, IEC, RoHS, FCC
- **Пища / agro:** EFSA, Codex Alimentarius, ТР ТС, СанПиН

## Что сохранять в результате

Файл: `<results_dir>/<topic>-fact-check-<YYYY-MM-DD>.md` (путь в `content_rules.fact_check.results_dir`).

```yaml
---
topic: "<тема>"
date: YYYY-MM-DD
checked_via: perplexity | codex | manual
total_claims: 12
confirmed: 9
needs_revision: 2
removed: 1
---

## Утверждения

### 1. "<текст утверждения>"
- **Статус:** confirmed | revised | removed
- **Источники:**
  - https://... — «короткая цитата»
  - https://... — «короткая цитата»
- **Финальная формулировка:** "..."
- **Применено в:** path/to/file.md:42
```

## Запреты (универсально)

- Не писать без источника: цифры (теплопроводность, плотность, ПДК, дозы), классы (горючесть, токсичность), сроки («срок службы 50 лет»).
- Не писать «общеизвестные факты» без проверки.
- Если линейка/модель/норматив снят с производства/устарел — не упоминать.
