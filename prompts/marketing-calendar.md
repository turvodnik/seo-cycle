# Единый маркетинг-календарь

Сводит ВСЕ активности в один план, чтобы каналы усиливали друг друга, а не жили
порознь: SEO-контент + соцсети + email + реклама + локалка + акции. `monthly-runner.sh`
ведёт SEO-контент; этот модуль — оркестрация всего маркетинга вокруг него.

## Что синхронизировать
| Поток | Источник | Ритм |
|---|---|---|
| Блог/категории (SEO) | `keyword-queue` + monthly-runner | напр. 1 статья/нед |
| Соцсети (VK/TG/Дзен) | `prompts/ad-and-social.md`, `distribution-channels.md` | 2-3 поста/нед |
| Карты-посты (Я.Бизнес/GBP) | `prompts/local/` | 2-3/нед |
| Email-рассылка | `marketing-skills:email-sequence` | 1-2/мес + триггерные |
| Реклама (если ROI>0) | `prompts/ad-and-social.md` + medi-план | постоянно, замер `roi-calc.py` |
| Акции/сезонность | сезонные пики (утеплители авг-окт) | по календарю спроса |

## Принцип связки
Каждая SEO-статья → анонс в VK/TG/Дзен → email-дайджест → (при акции) пост на картах +
объявление. Один инфоповод работает во всех каналах. Сезонные пики (Google Trends/
Wordstat) задают, что усиливать когда.

## Сборка
- 4-8-недельный календарь: дата | канал | тема/инфоповод | формат | статус | связанный SEO-URL.
- Связать с `keyword-queue` (контент) и approval-gates (перед публикацией).
- Приоритет инфоповодов → `ice-score.py`; частота vs конкуренты → `competitor-benchmark.py`.

## Выход
`<cycle>/marketing-calendar.md` — таблица на 4-8 недель по всем каналам. Раз в месяц
пересмотр (Phase 9-10): что вышло, что зашло (`roi-calc.py` / Метрика), корректировка.

## Автоматизация (опц.)
`monthly-runner.sh` уже ведёт еженедельный SEO-контент-поток с approval. Соцсети/email/
реклама пока ведутся по этому календарю вручную/полуавтоматически — при росте можно
вынести в отдельные потоки runner'а (но без избыточной автоматизации — см. заморозку дробления).
