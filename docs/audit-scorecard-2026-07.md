## Почему не «30 из 30 по 10»

Честность дороже красивой цифры:

- **[external]** — мир не даёт: Яндекс.Бизнес/2GIS открывают API только
  партнёрам; Google verification подаёт владелец аккаунта (пакет готов —
  `docs/gbp-verification-submission.md`); у Tilda нет write-API.
- **[policy]** — сами решили: GTM-изменения, ресабмит индексации, ads-apply
  и публикация всегда проходят человеческий approve. Автоматизировать «до 10»
  здесь значило бы убрать предохранитель, который продаёт агентство клиенту.
- **[осталось]** — почти пусто: автопроверка целей Метрики (14) и мелочи
  генерации вики (20). Woo-фиды, РСЯ-драфты, интервалы и срок жизни токенов
  закрыты в v1.86.

## Карта цикла (что стек делает для продвижения)

1. **Онбординг** — wizard, auth-профили (global/per-project), policy, doctor.
2. **Исследование** — семантика 6+ источников → кластеры (+SERP-overlap
   проверка) → сущности (+кандидаты из SERP) → evidence с живыми источниками.
3. **Стратегия** — структура (+визуальная карта), forecast, KPI-контракт,
   бюджет-микс SEO+PPC с убывающей отдачей.
4. **Продакшн** — брифы v3 → автоцикл качества → RAG-автоиндекс → публикация →
   индексация → repurpose в TG/VK/видео/email.
5. **Реклама** — Директ: fetch → кросс-аналитика (включая n-gram waste) →
   драфты → approve → apply; минус-слова и кандидаты в ядро.
6. **Мониторинг** — снапшоты по расписанию, прогресс позиций, краулер,
   живость ссылок, когорты Метрики, цитируемость в AI-ответах, триггеры.
7. **Отчётность** — самооценки 0–10 всех запусков, дашборд в браузере,
   white-label отчёт (md/HTML/PDF) с автоотправкой в Telegram, портфель
   агентства с пересечениями проектов.
8. **Контроль** — ledger/spend-guard, approvals, эскалации; человек решает,
   машина готовит и отчитывается.
