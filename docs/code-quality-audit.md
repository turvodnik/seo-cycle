Реальные баги, пойманные этой волной линта и боевыми прогонами:
1. `wiki-export-project-state.py` — `urlparse` без импорта, NameError глотался `except Exception` → ссылки с доменом проекта никогда не распознавались (v1.85).
2. `snapshot-build.py` — не понимал сырой формат Вебмастер API v4: 500 запросов молча схлопывались в пустые строки (v1.86.1, найден боевым прогоном на Эмвуди).
3. `client-report.py` — замыкание на переменные цикла в `fmt()` (B023): работало случайно, падало бы при любом рефакторинге порядка вызова (v1.87).
4. `snapshot-build.py` `_merge_snapshot` — engine хранился только в метаданных `sources`, строки запросов его не несли → в seo.db все позиции с пустым engine, per-engine фильтры молча вырождались. Найден первым же прогоном нового интеграционного теста (v1.88); живой снапшот Эмвуди пересобран.

## Что сделано в этой волне (v1.87)

- 18 bugbear-находок → 0: B023-замыкания (client-report), `raise … from`
  (gtm-audit, merchant-fetch), явные `strict=` у семи `zip()` (у cosine в RAG —
  `strict=True`: несовпадение длин векторов должно падать, а не молчать),
  7 неиспользуемых переменных цикла.
- `notify.py` переведён с собственной копии `load_env` на общую цепочку
  профилей (`env_profile.env_chain`) с поддержкой legacy `seo/.env`.
- Устранён последний локальный `/tmp`-hardcode (automation-plan).
- CI-линт ужесточён: `--select E9,F63,F7,F82,F401,F841,B,PLE`.

## Осознанный техдолг (не трогаем без причины)

1. **94 скрипта без core-импортов** — работают, самодостаточны; мигрируем при
   первом содержательном изменении файла, не ради галочки.
2. **Дубль парсера тикетов** в monthly-dashboard: формат стабилен, вынос в core
   потребует миграции авторского approval-gate — делать вместе с его следующей
   правкой.
3. **Отчётные скрипты без file-logging** — их stdout является контрактом;
   логировать им нечего.
4. **Браузерные коллекторы без тестов** — тестируется только их health-слой;
   сами сценарии проверяются работой.
